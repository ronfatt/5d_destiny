# Web 产品信息架构 V1

## 1. 文档定位

本文件定义五维命运系统 Web 平台的信息架构。

目标是把现有四大体系、卡牌系统、问卷系统、AI 报告系统转成可实现的网站结构。

本文件回答的问题是：

- 网站应该有哪些页面
- 每个页面承担什么职责
- 用户如何从进入到付费再到复访
- 数据和功能如何分层
- 哪些页面属于公开区，哪些属于会员区，哪些属于后台区

## 2. 产品目标

Web 平台必须同时承担四类任务：

- 获客
- 转化
- 交付
- 留存

因此，网站不能只做“算命页”，而必须是：

- 品牌入口
- 体验入口
- 支付入口
- 报告交付入口
- 会员复访入口

## 3. 架构总览

建议 Web 平台拆成五层：

1. 公开营销层
2. 体验转化层
3. 报告交付层
4. 会员留存层
5. 后台运营层

## 4. 一级站点结构

建议站点一级导航如下：

- 首页
- 五维系统
- 命运卡牌
- 报告与占卜
- 课程与认证
- 会员中心
- 登录 / 注册

说明：

- 若要极简，也可把“报告与占卜”作为主入口。
- 若要品牌化，“五维系统”必须保留，用于解释你不是普通塔罗站。

## 5. 公开营销层

### 5.1 首页

目标：

- 解释平台是什么
- 建立信任感
- 引导进入体验流程

首页建议模块：

1. Hero 区
2. 五维系统简介
3. 核心价值说明
4. 产品入口卡片
5. 卡牌视觉展示
6. 用户案例 / 评价
7. 价格与会员概览
8. CTA 区

关键按钮建议：

- `Start Destiny Reading`
- `Try Free Card Draw`
- `See Sample Report`

### 5.2 五维系统页

目标：

- 讲清楚方法论
- 解释五维变量
- 建立专业差异化

页面结构：

1. 什么是五维系统
2. 五维变量解释
3. 数学模型简述
4. 命盘、卡牌、问卷、AI 的关系
5. 示例分析

### 5.3 命运卡牌页

目标：

- 展示 50 张卡牌体系
- 建立视觉和 IP 识别度
- 引导抽牌体验与未来实体卡销售

页面结构：

1. 卡牌总览
2. 三类牌组说明
3. 精选卡牌展示
4. 抽牌玩法说明
5. 卡牌视觉系统说明

### 5.4 课程与认证页

目标：

- 承接对方法论感兴趣的用户
- 引导高价值用户进入教育产品

页面结构：

1. 课程体系总览
2. 四层/五层学习路径
3. 认证路线说明
4. 常见问题
5. 报名入口

## 6. 体验转化层

### 6.1 快速体验入口

目标：

- 让用户在最短路径内得到第一个结果
- 降低首次体验门槛

建议路径：

```text
输入问题 -> 抽牌 -> 生成轻量结果 -> 引导注册 / 升级
```

页面：

- 问题输入页
- 抽牌动画页
- 轻量结果页

### 6.2 标准占卜入口

目标：

- 采集完整输入
- 生成单主题标准报告

页面结构：

1. 基础信息页
2. 主题选择页
3. Mind / Action 问卷页
4. 抽牌页
5. 加载页
6. 标准报告页

### 6.3 深度报告入口

目标：

- 生成高价产品所需的完整输入
- 交付四主题深度报告

页面结构：

1. 基础信息页
2. 完整问卷页
3. 多主题配置页
4. 抽牌页
5. 支付页
6. 生成页
7. 完整报告页

## 7. 报告交付层

### 7.1 轻量结果页

适合：

- 免费抽牌
- 每日指数
- Bot 跳转页

输出内容：

- 一句话结论
- 当前卡牌
- 主导变量
- 一条建议
- 升级按钮

### 7.2 标准单主题报告页

适合：

- 单次付费占卜
- 单主题报告

输出结构：

1. 标题
2. 总体趋势
3. 五维拆解
4. 卡牌解析
5. 风险提醒
6. 行动建议
7. 升级完整版按钮

### 7.3 完整四主题报告页

适合：

- 年度报告
- 高价深度分析

输出结构：

1. 总览封面
2. 五维总说明
3. 四主题总览
4. 事业
5. 财富
6. 感情
7. 健康
8. 风险总表
9. 行动路线图

### 7.4 报告历史页

适合：

- 会员用户
- 复访用户

功能：

- 查看历史报告
- 对比不同时间结果
- 重新生成或更新问卷

## 8. 会员留存层

### 8.1 会员中心首页

目标：

- 成为复访中枢
- 呈现用户长期关系与数据沉淀

建议模块：

- 今日命运指数
- 本周变化
- 最近抽牌
- 最新报告
- 问卷更新提醒
- 会员权益区

### 8.2 每日指数页

输出：

- 今日总指数
- 今日主导变量
- 今日卡牌提示
- 今日一句行动建议

### 8.3 周期更新页

输出：

- 本周趋势
- 本月趋势
- 变化对比图
- 建议重点

### 8.4 会员权益页

展示：

- 当前套餐
- 已解锁功能
- 升级路径
- 续费状态

## 9. 后台运营层

### 9.1 用户管理

功能：

- 用户基本信息
- 订阅状态
- 报告历史
- 问卷记录

### 9.2 报告管理

功能：

- 查看生成结果
- 检查异常报告
- 标记质量问题
- 导出报告数据

### 9.3 卡牌管理

功能：

- 卡牌数据维护
- Prompt 版本维护
- 主题偏向值维护

### 9.4 课程管理

功能：

- 课程上架
- 学员记录
- 认证状态
- 案例作业管理

## 10. 页面流转设计

### 10.1 首次用户主路径

```text
首页
-> 免费体验
-> 轻量结果
-> 注册
-> 标准报告
-> 会员转化
```

### 10.2 付费用户路径

```text
首页 / 广告页
-> 标准输入
-> 支付
-> 抽牌
-> 报告生成
-> 报告页
-> 会员升级
```

### 10.3 会员用户路径

```text
登录
-> 会员中心
-> 今日指数
-> 周期报告
-> 历史报告对比
-> 课程 / 咨询入口
```

## 11. 核心数据对象

Web 平台至少应围绕以下对象设计：

- User
- BirthProfile
- DestinyReading
- CardDraw
- QuestionnaireResult
- DestinyReport
- Membership
- CourseEnrollment

### 11.1 DestinyReading

建议包含：

- 问题
- 主题
- 时间戳
- 抽牌结果
- 五维输入
- 五维得分
- 报告引用

### 11.2 DestinyReport

建议包含：

- 报告类型
- 生成时间
- 版本号
- 文本结果
- JSON 结果
- 是否可导出 PDF

4. 问卷页
5. 抽牌页
6. 单主题报告页

## 15. 当前版本结论

Web 信息架构的作用是把五维系统从“概念 + 引擎 + 文档”推进到真正的网站产品结构。

如果没有这份文档，后面做前端时会很容易出现：

- 页面重复
- 路径混乱
- 会员逻辑断裂
- 报告与课程入口脱节

有了这份 IA，下一步如果要继续，就可以直接进入：

- 前端页面清单
- 站点地图
- 低保真线框图
- 数据库表设计
