# 五维命运系统 v1 主题问卷设计表

## 1. 文档目的

本文件用于定义五维命运系统在前端表单、AI 对话、会员评估流程中的问卷输入结构。

问卷层负责补足两类变量：

- `Mind`：心念与认知状态
- `Action`：现实推进与执行强度

本文件不负责最终参数边界，参数边界以 [five-dimension-destiny-system-v1-parameter-table.md](/Users/rms/Desktop/Ai Project/Ai5Ddestiny/docs/five-dimension-destiny-system-v1-parameter-table.md) 为准。

## 2. 问卷设计原则

### 2.1 可评分

每一道题都必须可映射到明确分值，不允许只产生模糊叙事。

### 2.2 可主题化

同样的 Action 变量，在事业、财富、感情、健康中观察项不同。

### 2.3 可对话化

题目既能用于表单，也能用于 AI 问答。

### 2.4 可分层

问卷输出先形成中间字段，再映射到五维变量，不直接碰最终趋势分。

## 3. 输入结构总览

问卷输出分为两层：

- Mind Survey Output
- Action Survey Output

建议接口结构：

```json
{
  "theme": "career",
  "mindSurvey": {
    "answers": [6, 5, 6, 4, 5, 5, 6, 5, 6, 5],
    "average": 5.3,
    "mindLevel": 5,
    "mindFactor": 1.06
  },
  "actionSurvey": {
    "answers": [2, 2, 1, 1, 2],
    "total": 8,
    "actionRaw": 72,
    "actionLevel": "稳定执行"
  }
}
```

## 4. Mind 问卷设计

### 4.1 目标

Mind 问卷用于评估：

- 认知清晰度
- 情绪稳定度
- 主体性
- 长期导向
- 责任承担

### 4.2 评分方式

- 题量：`10 题`
- 每题分值：`1-7`
- 结果：取平均值 `MindSurveyAvg`

### 4.3 题目设计

1. 遇到问题时，你更倾向立刻面对还是先逃避。
2. 做决定时，你更依赖清晰判断还是情绪波动。
3. 当事情不顺时，你更容易自我否定还是主动修正。
4. 你是否知道当前人生阶段最重要的目标。
5. 你是否清楚自己下一步最该做什么。
6. 你是否容易把恐惧误判为现实。
7. 你能否持续把注意力放在长期价值上。
8. 你在压力下是否仍能保持基本稳定。
9. 你更常被外界推着走，还是主动创造节奏。
10. 你是否对自己的选择承担责任并持续校正。

### 4.4 选项提示建议

为了便于前端实现，建议统一使用 7 级选项：

- `1`：非常不符合
- `2`：不太符合
- `3`：略不符合
- `4`：一般
- `5`：较符合
- `6`：很符合
- `7`：非常符合

### 4.5 结果映射

| 平均分 | Mind Level | 引擎系数 |
| --- | --- | --- |
| 1.0-1.9 | 1 | 0.82 |
| 2.0-2.9 | 2 | 0.88 |
| 3.0-3.9 | 3 | 0.94 |
| 4.0-4.9 | 4 | 1.00 |
| 5.0-5.4 | 5 | 1.06 |
| 5.5-6.2 | 6 | 1.12 |
| 6.3-7.0 | 7 | 1.18 |

### 4.6 AI 对话评分建议

若由 AI 问答生成评分，建议流程：

1. AI 提问并记录用户原始表达
2. 将表达归类到 10 个 Mind 题项
3. 每题产出 `1-7` 分
4. 输出总平均与对应系数
5. 若信心低，标记 `low_confidence`

建议返回结构：

```json
{
  "mindSurvey": {
    "answers": [5, 6, 4, 5, 5, 4, 6, 5, 5, 6],
    "average": 5.1,
    "mindLevel": 5,
    "mindFactor": 1.06,
    "confidence": 0.78
  }
}
```

## 5. Action 问卷设计

### 5.1 目标

Action 问卷用于评估用户在具体主题上的现实推进强度。

Action 关注：

- 是否有实际动作
- 是否稳定持续
- 是否形成结果
- 是否具有高杠杆价值

### 5.2 评分方式

- 每个主题：`5 题`
- 每题分值：`0-2`
- 总分范围：`0-10`
- 再映射到 `ActionRaw 0-100`

### 5.3 通用评分规则

- `0`：几乎没有行为
- `1`：有行为但不稳定、不完整或价值有限
- `2`：稳定、明确、可验证、有效

## 6. 四大主题 Action 问卷

### 6.1 事业问卷

1. 最近 30 天是否持续推进核心目标。
2. 是否有明确项目、作品、提案或交付输出。
3. 是否主动建立或维护关键资源连接。
4. 是否记录并复盘自己的推进结果。
5. 是否真正执行，而不是长期停留在构想层。

### 6.2 财富问卷

1. 是否有稳定的现金流管理动作。
2. 是否持续记账、储蓄或控制支出。
3. 是否推进真正影响收入的动作。
4. 是否有明确的成交、变现或资产积累行为。
5. 是否避免高频冲动消费或混乱投资。

### 6.3 感情问卷

1. 是否有稳定且真实的沟通。
2. 是否在关系中提供回应与经营投入。
3. 是否清楚表达边界与需求。
4. 是否愿意处理问题而不是逃避或冷处理。
5. 是否用建设性互动替代情绪化消耗。

### 6.4 健康问卷

1. 是否保持基本规律作息。
2. 是否执行运动、饮食或恢复计划。
3. 是否处理已经知道的健康问题。
4. 是否减少明显透支行为。
5. 是否有检查、治疗或恢复跟进。

## 8. 前端实现建议

### 8.1 最低可用版本

前端最简版只需要：

- 1 个主题选择器
- 10 道 Mind 题
- 5 道对应主题的 Action 题
- 自动输出 Mind 系数与 Action 原始值

### 8.2 结果层字段

前端问卷提交后，建议直接产出以下字段：

```json
{
  "theme": "relationship",
  "mindFactor": 1.00,
  "mindLevel": 4,
  "actionRaw": 72,
  "actionLabel": "稳定执行"
}
```

### 8.3 AI 辅助模式

当用户不想填表时，可由 AI 通过对话采集：

- 先问 3-5 个开放问题
- 再由系统自动映射到 Mind / Action
- 最后返回一个“系统理解结果”，供用户确认

## 9. 风险控制

### 9.1 自评偏差

用户可能高估或低估自己，因此建议：

- Mind 问卷与 Action 问卷都支持 AI 复核
- 系统对明显矛盾回答进行提示

### 9.2 主题错配

例如用户在做“财富问卷”时，实际回答成“事业动作”。

建议：

- 每一题都在 UI 中明确主题上下文
- AI 问答模式中先复述当前主题

### 9.3 长期漂移

用户不同时间填写，结果会波动。

这是系统特性，不是错误。

建议：

- 保存历史问卷记录
- 允许用户比较 7 天、30 天、90 天变化
