# 50 张命运卡数据表 V1

## 1. 文档定位

本文件是第二体系《Destiny Card System V1》的数据层补充文档。

目标是把卡牌说明书中的叙述内容，整理成可直接服务以下模块的结构化档案：

- 抽牌引擎
- Energy 计算引擎
- AI 解读引擎
- 卡牌手册
- 后续实体卡或数字卡设计

本文件优先服务数据实现，因此采用“逐张卡片档案”格式，而不是散文式说明。

## 2. 字段标准

每张卡建议至少包含以下字段：

- `id`
- `name`
- `category`
- `valueMode`
- `baseValue` 或 `positiveValue/negativeValue`
- `keywords`
- `positiveMeaning`
- `shadowMeaning`
- `interpretationHint`

字段说明：

- `category`：`archetype / energy / event`
- `valueMode`：`fixed / dual`
- `fixed` 表示固定分值
- `dual` 表示双极卡，需要按语境判断正负方向

- `1` 张事件触发卡

原始计算：

`CardEnergyRaw = Archetype × 0.30 + Energy1 × 0.25 + Energy2 × 0.25 + Event × 0.20`

映射到系统 Energy：

`Energy = clamp(CardEnergyRaw × 3, -20, 20)`

## 7. 解读顺序标准

建议 AI 与人工解读统一采用以下顺序：

1. 主星卡：事情背后的核心驱动力
2. 能量卡 1：当前主要状态
3. 能量卡 2：辅助状态或阻力
4. 事件卡：现实触发点

示例：

- 破军革命
- 机会
- 压力
- 合作

标准解读：

- 破军：正在破局，旧结构不再适用
- 机会：新的入口已经出现
- 压力：推进过程中负担明显
- 合作：必须借助伙伴或团队，不宜单干

结论：

- 高风险机会型局面
