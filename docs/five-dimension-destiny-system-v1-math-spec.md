# 五维命运系统 v1.0 数学模型规格书

## 1. 文档目的

本规格书用于定义“五维命运系统”的第一版可执行评分逻辑，作为后续以下模块的统一底层标准：

- 命理规则引擎
- 命运卡牌系统
- AI 解读系统
- 前端报告系统
- 课程与案例体系

本版本目标不是证明“绝对预测”，而是建立一套可重复、可解释、可校准的趋势评估模型。

## 2. 系统定义

五维命运系统将命运趋势视为五类变量的综合结果：

- Structure：命格结构，代表长期底层配置
- Timing：时运周期，代表阶段性时间窗口
- Energy：当下能量，代表短期状态扰动
- Mind：心念频率，代表内在导向与认知状态
- Action：行动变量，代表现实执行强度

系统不输出“注定结果”，只输出“趋势结果、主导变量、风险项、建议项”。

## 3. 设计原则

### 3.1 可解释

每个分数都必须能回溯到输入来源与规则映射，不能只给结果不给理由。

### 3.2 可分层

系统采用三层计算结构：

- 原始维度层
- 中间结果层
- 最终输出层

### 3.3 可校准

权重、阈值、文本模板允许随着真实用户反馈迭代。

### 3.4 可扩展

本版本先支持四个主题域：

- 事业
- 财富
- 感情
- 健康

后续可扩展至学业、人际、家庭、灵性等主题域。

## 4. 五维变量定义

### 4.1 Structure

定义：个体在某一主题上的长期底层结构强度。

来源建议：

- 紫微斗数命盘主结构
- 主题相关宫位
- 主星、辅星、四化、格局等规则映射

取值范围：`0-100`

解释区间：

- `0-39`：结构薄弱
- `40-59`：结构普通
- `60-79`：结构良好
- `80-100`：结构突出

示例：

- 事业结构：82
- 财富结构：65
- 感情结构：48
- 健康结构：71

### 4.2 Timing

定义：当前时间周期对某主题产生的放大或压缩效应。

来源建议：

- 大运
- 流年
- 流月

取值形式：`倍率系数`

取值范围：`0.70-1.30`

解释区间：

- `0.70-0.84`：明显逆风期
- `0.85-0.94`：偏弱期
- `0.95-1.05`：平稳期
- `1.06-1.15`：上升期
- `1.16-1.30`：强势窗口

说明：

- Timing 不建议使用过大倍率，避免结果失真。
- v1.0 先将 Timing 作为线性乘数，而不是复合乘方项。

### 4.3 Energy

定义：用户当前阶段的能量状态与环境氛围。

来源建议：

- 三张命运卡 / 塔罗三张阵
- 近期状态问答
- 情绪与状态标签

取值范围：`-20 至 +20`

解释区间：

- `-20 至 -11`：低迷阻滞
- `-10 至 -4`：耗损偏重
- `-3 至 +3`：中性波动
- `+4 至 +10`：能量顺畅
- `+11 至 +20`：高能推进

说明：

Energy 是短期扰动项，不代表长期命格，仅代表当前状态修正。

### 4.4 Mind

定义：用户当前内在频率、认知质量与心态导向。

来源建议：

- 频率问卷
- 心理状态量表
- 自我叙述标签
- 数字频率模型

取值形式：`调节系数`

取值范围：`0.80-1.20`

解释区间：

- `0.80-0.89`：恐惧/混乱/自耗
- `0.90-0.97`：焦虑/犹豫/分散
- `0.98-1.03`：中性/观望
- `1.04-1.11`：平静/清晰/创造
- `1.12-1.20`：使命/聚焦/高一致性

说明：

Mind 不直接决定命运结果，但会影响能量转化与行动落地效率。

### 4.5 Action

定义：用户在目标主题上的实际推进强度。

来源建议：

- 行动频率问卷
- 最近 30 天行为记录
- 执行承诺完成率

取值范围：`0-100`

解释区间：

- `0-19`：停滞
- `20-39`：低执行
- `40-59`：基本执行
- `60-79`：稳定推进
- `80-100`：高强度推进

说明：

v1.0 将 Action 设计为独立主变量，而不是尾部装饰项，以确保“可改变性”在模型中真实存在。

## 5. 标准化原则

五维变量量纲不同，必须先标准化后再参与部分计算。

### 5.1 标准化公式

- `S = Structure / 100`
- `T = Timing`
- `E = 1 + (Energy / 100)`
- `M = Mind`
- `A = Action / 100`

标准化后范围约为：

- `S`: `0.00-1.00`
- `T`: `0.70-1.30`
- `E`: `0.80-1.20`
- `M`: `0.80-1.20`
- `A`: `0.00-1.00`

说明：

- 将 `Energy` 转为 `1 + Energy/100`，是为了避免负值直接破坏整体结构。
- `Action` 标准化后可与 `Mind` 形成独立执行项。

## 6. 三层计算结构

### 6.1 原始维度层

原始输入：

- Structure
- Timing
- Energy
- Mind
- Action

### 6.2 中间结果层

定义三个中间指标：

#### Base Potential

表示基础趋势潜力：

`BasePotential = S × T`

#### State Momentum

表示当前状态与心念带来的推进或阻滞：

`StateMomentum = E × M`

#### Execution Power

表示行动与心念的一致性所产生的现实转化能力：

`ExecutionPower = A × M`

### 6.3 最终输出层

v1.0 总分公式：

`TrendIndexRaw = (BasePotential × 50) + (StateMomentum × 20) + (ExecutionPower × 30)`

`TrendScore = round(TrendIndexRaw)`

说明：

- 三项权重总计 100。
- 结构与时运合计权重最高，保持命理底盘的主导性。
- 行动项权重为 30，确保结果可被用户主动改变。
- 状态项权重为 20，避免短期情绪过度放大结果。

## 7. 结果等级定义

### 7.1 TrendScore 区间

- `0-39`：高风险回撤
- `40-54`：弱势整理
- `55-69`：平稳积累
- `70-84`：积极上升
- `85-100`：强势增长
- `100+`：突破窗口

说明：

- 分数允许超过 100，用于标识极强窗口。
- 前端显示时可同时展示原始分与等级标签。

### 7.2 风险判定规则

满足任一条件时，标记为重点风险：

- `Timing < 0.90`
- `Energy <= -10`
- `Mind <= 0.90`
- `Action < 30`
- `Structure < 40`

### 7.3 关键变量判定规则

计算各项对总分的边际影响，取绝对值最高者作为主导变量。

简化版规则：

- 若 `Timing` 偏离 `1.00` 最大，则主导变量为时运
- 若 `Energy` 绝对值最大，则主导变量为能量
- 若 `Mind` 与 `Action` 同时较高或较低，则主导变量为执行
- 若 `Structure >= 80` 或 `Structure <= 40`，则结构可直接成为主导变量

## 8. 主题域分公式

v1.0 支持四个主题域，每个主题域使用同一总公式，但允许来源与权重映射不同。

### 8.1 事业主题

输入来源：

- Structure: 命宫、官禄宫、身宫、迁移宫
- Timing: 事业相关大运、流年、流月
- Energy: 事业牌阵/当前工作状态
- Mind: 清晰度、责任感、创造状态
- Action: 投递、提案、执行、协作推进度

事业总分：

`CareerScore = TrendScore`

### 8.2 财富主题

输入来源：

- Structure: 财帛宫、田宅宫、福德宫
- Timing: 财富相关流运指标
- Energy: 财富能量牌阵
- Mind: 匮乏/丰盛心态、金钱认知
- Action: 销售、储蓄、投资纪律、成交行为

财富总分：

`WealthScore = TrendScore`

### 8.3 感情主题

输入来源：

- Structure: 夫妻宫、命宫、福德宫、迁移宫
- Timing: 感情相关流运指标
- Energy: 关系牌阵
- Mind: 安全感、表达能力、情绪稳定度
- Action: 沟通、回应、边界、经营投入

感情总分：

`RelationshipScore = TrendScore`

### 8.4 健康主题

输入来源：

- Structure: 疾厄宫、福德宫、命宫
- Timing: 健康相关流运指标
- Energy: 身心状态牌阵
- Mind: 压力、恢复感、睡眠与精神张力
- Action: 作息、运动、饮食、检查与恢复执行

健康总分：

`HealthScore = TrendScore`

## 9. 输出模板结构

每个主题域报告统一输出以下内容：

### 9.1 总体趋势

根据 `TrendScore` 输出：

- 高风险回撤
- 弱势整理
- 平稳积累
- 积极上升
- 强势增长
- 突破窗口

### 9.2 主导变量

从以下标签中选择一项：

- 结构主导
- 时运主导
- 能量主导
- 心念主导
- 行动主导
- 执行协同主导

### 9.3 风险提示

从规则中抽取 1-3 项：

- 时机未开，不宜强攻
- 状态耗损，判断易失真
- 心念分散，执行折损
- 结构一般，需要借势而行
- 行动力不足，错失窗口

### 9.4 行动建议

建议必须与 Action 相关，不能只给抽象安慰。

输出格式建议：

- 未来 7 天行动建议
- 未来 30 天调整重点
- 当前最优策略

## 10. 文本模板映射规则

### 10.1 高结构 + 低时运

条件：

- `Structure >= 75`
- `Timing <= 0.92`

输出倾向：

- 底子强，但窗口未开
- 当前宜蓄势，不宜焦躁冒进

### 10.2 中结构 + 高行动

条件：

- `Structure 45-65`
- `Action >= 70`

输出倾向：

- 先天普通，但后天可追
- 结果增量主要来自执行

### 10.3 高能量 + 低心念

条件：

- `Energy >= 10`
- `Mind <= 0.92`

输出倾向：

- 外部推动存在，但内在不稳定
- 易冲动决策，需要先校准方向

### 10.4 低能量 + 高行动

条件：

- `Energy <= -8`
- `Action >= 65`

输出倾向：

- 执行持续，但恢复不足
- 需防止硬撑导致反噬

## 11. 样例演算

### 11.1 案例 A：事业上升型

输入：

- Structure = 82
- Timing = 1.12
- Energy = 8
- Mind = 1.08
- Action = 76

标准化：

- `S = 0.82`
- `T = 1.12`
- `E = 1.08`
- `M = 1.08`
- `A = 0.76`

中间计算：

- `BasePotential = 0.82 × 1.12 = 0.9184`
- `StateMomentum = 1.08 × 1.08 = 1.1664`
- `ExecutionPower = 0.76 × 1.08 = 0.8208`

总分：

- `TrendIndexRaw = (0.9184 × 50) + (1.1664 × 20) + (0.8208 × 30)`
- `TrendIndexRaw = 45.92 + 23.33 + 24.62 = 93.87`
- `TrendScore = 94`

结果：

- 趋势：强势增长
- 主导变量：结构主导
- 风险：节奏过快时可能出现透支
- 建议：适合推进关键项目与资源整合

### 11.2 案例 B：财富平稳型

输入：

- Structure = 63
- Timing = 1.01
- Energy = 2
- Mind = 1.00
- Action = 51

标准化：

- `S = 0.63`
- `T = 1.01`
- `E = 1.02`
- `M = 1.00`
- `A = 0.51`

中间计算：

- `BasePotential = 0.6363`
- `StateMomentum = 1.02`
- `ExecutionPower = 0.51`

总分：

- `TrendIndexRaw = (0.6363 × 50) + (1.02 × 20) + (0.51 × 30)`
- `TrendIndexRaw = 31.82 + 20.40 + 15.30 = 67.52`
- `TrendScore = 68`

结果：

- 趋势：平稳积累
- 主导变量：行动主导
- 风险：容易因观望过久错过机会
- 建议：保持现金流纪律，逐步扩大有效行动

### 11.3 案例 C：感情回撤型

输入：

- Structure = 48
- Timing = 0.86
- Energy = -9
- Mind = 0.91
- Action = 28

标准化：

- `S = 0.48`
- `T = 0.86`
- `E = 0.91`
- `M = 0.91`
- `A = 0.28`

中间计算：

- `BasePotential = 0.4128`
- `StateMomentum = 0.8281`
- `ExecutionPower = 0.2548`

总分：

- `TrendIndexRaw = (0.4128 × 50) + (0.8281 × 20) + (0.2548 × 30)`
- `TrendIndexRaw = 20.64 + 16.56 + 7.64 = 44.84`
- `TrendScore = 45`

结果：

- 趋势：弱势整理
- 主导变量：时运主导
- 风险：情绪误判与低投入共同压低结果
- 建议：当前以修复节奏与边界重建为主，不宜强求推进

## 14. 当前版本限制

v1.0 暂不处理以下问题：

- 紫微斗数内部复杂星曜组合的精细权重
- 多牌阵之间的冲突判定
- 用户长期行为数据自动回填
- 多主题域之间的相互影响
- 非线性极端事件预测

这些内容应放入 v1.1 或 v2.0 迭代。
